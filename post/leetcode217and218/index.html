Leetcode-217&&219.存在重复元素(Contains Duplicate)相关

# Sep 18, 2019

采用语言为Java

  • LeetCode国内官网：https://leetcode-cn.com

  • 因为题号和题名都有，题目描述就不写了

简单难度

217.存在重复元素(Contains Duplicate)

解法一

如果我们直接遍历做这道题，时间复杂度会有O(n^2)，我们可以先对数组排个序就可以通过了

class Solution {
    public boolean containsDuplicate(int[] nums) {
        Set<Integer> set = new HashSet<>(nums.length);
        for (int x: nums) {
            if (set.contains(x))
                return true;
            set.add(x);
        }
        return false;
    }
}

执行用时：9ms 内存消耗：48MB

解法二

用了哈希表，在线性的时间复杂度就可以解决，记住这个点，以后能用就用

class Solution {
    public boolean containsDuplicate(int[] nums) {
        Set<Integer> set = new HashSet<>(nums.length);
        for (int x: nums) {
            if (set.contains(x))
                return true;
            set.add(x);
        }
        return false;
    }
}

执行用时：14ms 内存消耗：44.5MB

219.存在重复元素II(Contains Duplicate II)

用到了HashMap,因为有key和value，个人觉得还挺好理解~要注意的一点就是存HashMap的时候，并不是按照i:nums[i]存的，而是把nums[i]存为key，i存为value，因为自带的get方法可以通过传入一个key来获取value，如果这样存就可以很方便的获得它的i值从而进行索引值i，j的计算，来达到题目要求。

class Solution {
    public boolean containsNearbyDuplicate(int[] nums, int k) {
        Map<Integer, Integer> map = new HashMap<>();

        for (int i = 0; i < nums.length; i++) {
            if (!map.containsKey(nums[i])) {
                map.put(nums[i], i);
            } else {
                int j = map.get(nums[i]);
                if (Math.abs(i - j) <= k)
                    return true;
                map.put(nums[i], i);
            }

        }
        return false;
    }
}

执行用时：29ms 内存消耗：49.1MB
