Leetcode-83.删除排序链表中的重复元素(Remove Duplicates from Sorted List)

简单难度

83.删除排序链表中的重复元素(Remove Duplicates from Sorted List)

参考了某大佬思路，题目要求是删除重复元素，首先判断链表是否为空或者链表是否只剩一个元素，这种情况直接返回就可以了；接下来进行递归操作，判断头节点值是否等于头节点后next节点的值，相等表明该元素重复，返回head.next值即可

/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) { val = x; }
 * }
 */
class Solution {
    public ListNode deleteDuplicates(ListNode head) {
        if (head == null || head.next == null)
            return head;
        head.next = deleteDuplicates(head.next);
        if (head.val == head.next.val)
            head = head.next;
        return head;
    }
}

执行用时：2ms 内存消耗：37.4MB

Leetcode-21.合并两个有序链表(Merge Two Sorted Lists)

简单难度

21.合并两个有序链表(Merge Two Sorted Lists)

这道题目是一个有关链表的题目中，注释中给出了ListNode的定义

整个合并过程可以看作是一个递归的过程，两个链表中头部较小的一个与剩下的元素进行递归合并

/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) { val = x; }
 * }
 */
class Solution {
    public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
        if (l1 == null) {
            return l2;
        } else if (l2 == null) {
            return l1;
        } else if (l1.val < l2.val) {
            l1.next = mergeTwoLists(l1.next, l2);
            return l1;
        } else {
            l2.next = mergeTwoLists(l1, l2.next);
            return l2;
        }
    }
}

执行用时：1ms 内存消耗：36.6MB

简单难度

217.存在重复元素(Contains Duplicate)

解法一

如果我们直接遍历做这道题，时间复杂度会有O(n^2)，我们可以先对数组排个序就可以通过了

class Solution {
    public boolean containsDuplicate(int[] nums) {
        Set<Integer> set = new HashSet<>(nums.length);
        for (int x: nums) {
            if (set.contains(x))
                return true;
            set.add(x);
        }
        return false;
    }
}

执行用时：9ms 内存消耗：48MB

解法二

用了哈希表，在线性的时间复杂度就可以解决，记住这个点，以后能用就用

执行用时：14ms 内存消耗：44.5MB

219.存在重复元素II(Contains Duplicate II)

用到了HashMap,因为有key和value，个人觉得还挺好理解~要注意的一点就是存HashMap的时候，并不是按照i:nums[i]存的，而是把nums[i]存为key，i存为value，因为自带的get方法可以通过传入一个key来获取value，如果这样存就可以很方便的获得它的i值从而进行索引值i，j的计算，来达到题目要求。

class Solution {
    public boolean containsNearbyDuplicate(int[] nums, int k) {
        Map<Integer, Integer> map = new HashMap<>();

        for (int i = 0; i < nums.length; i++) {
            if (!map.containsKey(nums[i])) {
                map.put(nums[i], i);
            } else {
                int j = map.get(nums[i]);
                if (Math.abs(i - j) <= k)
                    return true;
                map.put(nums[i], i);
            }

        }
        return false;
    }
}

执行用时：29ms 内存消耗：49.1MB
